Kalman Filters

Task Description

The task is to track a car using LIDAR and RADAR sensors. Each of the sensors has its strengths and weaknesses. We will use Kalman Filter to fuse data from both.

Our sensor team did all necessary filtering and tracking of LIDAR point clouds and RADAR data, and provides us with [x,y]T[x, y]^T[x,y]T position from LIDAR and [ρ,ϕ,ρ˙]T[\rho, \phi, \dot \rho]^T[ρ,ϕ,ρ˙​]T data from RADAR.

LIDAR

LIght Detection And Ranging - sensor that emits laser rays and returns a point cloud with distances. This video explains how LIDAR works.

In our case we assume we have detected objects in a scene and represened each with a bounding box, ID, and estimated location. How will it look like? Check this video.

For simplicity we assume only one object - car we want to track. So, data from LIDAR will have the form:

yL=[pxpy]y_L = \begin{bmatrix} p_x \\ p_y\end{bmatrix}yL​=[px​py​​]

RADAR

RAdio Detection And Ranging - sensor that uses Dopler effect to measure velocity and position. The ability of RADAR to independently measure velocity is very important for us. Also, RADARs perform better than LIRARs in bad weather conditions. This video explains how RADAR works.

The only problem is RADAR returns its data in Polar form, and we will need to convert it into Cartesian system.

So, data from RADAR will have the form:

yR=[ρϕρ˙]y_R = \begin{bmatrix} \rho \\ \phi \\ \dot \rho\end{bmatrix}yR​=⎣⎡​ρϕρ˙​​⎦⎤​

What are www and vvv

www - process noise. Includes everything that our model does not take into account. Friction, sudden wind, slipping, etc. Often represented as a multivariate Gaussian distribution with zero mean and covariance matrix WWW.

vvv - measurement noise. Our sensors are not ideal, measurements contain some noise that we have to deal with. Often represented as a multivariate Gaussian distribution with zero mean and covariance matrix VVV. Sensor manufacturer should provide VVV for you, or you can calculate it yourself.

Kalman filter

We estimate x^\hat xx^ in two steps:

  1. Dynamic update (also called Prediction)
    x^i∣i−1\hat x_{i|i-1}x^i∣i−1​ - is the i-th step estimate after dynamics update; also called an a priori estimate.
  2. Sensor update (also called Measurement update)
    x^i∣i\hat x_{i|i}x^i∣i​ - is the estimate after sensor update; also called an a posteriori estimate

From now on I will use sligtly different notation for x^i∣i\hat x_{i|i}x^i∣i​ and x^i∣i−1\hat x_{i|i-1}x^i∣i−1​. Since we always use previous estimated state to predict the next one, iii notation is redundant. So, x^i∣i\hat x_{i|i}x^i∣i​ becomes x^\hat xx^, and x^i∣i−1\hat x_{i|i-1}x^i∣i−1​ becomes x^′\hat x'x^′

With new notation Kalman filter equations become:

  1. Prediction
    x^′=Ax^+Bu\hat x' = A\hat x + Bux^′=Ax^+Bu
    P′=APAT+WP' = APA^{T}+WP′=APAT+W
  2. Measurement
    Y=CP′CT+VY = CP'C^{T} + VY=CP′CT+V
    K=P′CTY−1K = P'C^{T}Y^{-1}K=P′CTY−1
    x^=x^′+K(y−Cx^′)\hat x = \hat x' + K(y - C\hat x')x^=x^′+K(y−Cx^′)
    P=(I−KC)P′P = (I - KC)P'P=(I−KC)P′

In our situation we will use Kalman Filter to track 2D position and velocity of a car using only noisy LIDAR measurements. The cool thing is - Kalman Filter will handle noise. Another cool thing - we can estimate velocity of the car by only measuring its position.

Extended Kalman filter

xxx and yyy are functions in general: x′=f(x,u)x' = f(x, u)x′=f(x,u), y=g(x′)y=g(x')y=g(x′) . Extended Kalman filter is used in case of nonlinearities in fff or ggg. This means, we cannot represent state transition and input matrces as matrices. Why nonlinear is bad for us? Check Appendix. What we can do, is we can use Taylor expansion to approximate fff and ggg.

Taylor expansion reminder
Assume we have scalar function f(x) and we want to approximate it with a linear function at point μ\muμ. The Taylor expansion will look like this:

f(x)≈f(μ)+∂f(μ)∂x(x−μ)f(x) \approx f(\mu) + \frac{\partial f(\mu)}{\partial x}(x-\mu)f(x)≈f(μ)+∂x∂f(μ)​(x−μ)

We can also expand this definition to vectors:

T(x)=f(μ)+(x−μ)TDf(μ)+(x−μ)TD2f(μ)(x−μ)+...T(x) = f(\mu) + (x-\mu)^{T}Df(\mu) + (x-\mu)^{T}D^2f(\mu)(x-\mu) + ...T(x)=f(μ)+(x−μ)TDf(μ)+(x−μ)TD2f(μ)(x−μ)+...

Df(μ)Df(\mu)Df(μ) is called the Jacobian matrix and D2f(μ)D^2f(\mu)D2f(μ) is called the Hessian matrix. They represent first order and second order derivatives of multi-dimensional equations. A full Taylor series expansion would include higher order terms as well for the third order derivatives, fourth order derivatives, and so on.

Updated Kalman filter equations look like this:

  1. Prediction
    x^′=f(x^,u)\hat x' = f(\hat x, u)x^′=f(x^,u)
    P′=AjPAjT+WP' = A_{j}PA_{j}^{T}+WP′=Aj​PAjT​+W
  2. Measurement
    Y=CjP′CjT+VY = C_{j}P'C_{j}^{T} + VY=Cj​P′CjT​+V
    K=P′CjTY−1K = P'C_{j}^{T}Y^{-1}K=P′CjT​Y−1
    x^=x^′+K(y−g(x^′))\hat x = \hat x' + K(y - g(\hat x'))x^=x^′+K(y−g(x^′))
    P=(I−KCj)P′P = (I - KC_{j})P'P=(I−KCj​)P′

In our case f(x^,u)f(\hat x, u)f(x^,u) is linear, and can be represented with AAA matrix. So, Aj=AA_{j} = AAj​=A.

g(x^′)g(\hat x')g(x^′) is more interesting. To derive a linear approximation for the RADAR’s g(x^′)g(\hat x')g(x^′) function, we will only keep the expansion up to the Jacobian matrix Df(μ)Df(\mu)Df(μ). We will ignore the Hessian matrix D2f(μ)D^2f(\mu)D2f(μ) and other higher order terms. Assuming (x−μ)(x−μ)(x - \mu)(x-\mu)(x−μ)(x−μ) is small, (x−μ)2(x - \mu)^2(x−μ)2 or the multi-dimensional equivalent (x−μ)T(x−μ)(x-\mu)^T (x - \mu)(x−μ)T(x−μ) will be even smaller; the extended Kalman filter we’ll be using assumes that higher order terms beyond the Jacobian are negligible.

In our case we have to calculate this expression:

Cj=∂CR(x^)∂x^C_j = \frac{\partial C_R(\hat x)}{\partial \hat x}Cj​=∂x^∂CR​(x^)​

Ok, we need to calculate some partial derivatives. Here is the result:

Cj=[pxpx2+py2pypx2+py200−pypx2+py2pxpx2+py200py(vxpy−vypx)(px2+py2)3/2px(vypx−vxpy)(px2+py2)3/2pxpx2+py2pypx2+py2]C_j = \begin{bmatrix} \frac{p_x}{\sqrt{p_x^2+p_y^2} } & \frac{p_y}{\sqrt{p_x^2+p_y^2} } & 0 & 0 \\- \frac{p_y}{p_x^2+p_y^2 } & \frac{p_x}{p_x^2+p_y^2 } & 0 & 0 \\ \frac{p_y(v_xp_y-v_yp_x)}{(p_x^2+p_y^2)^{3/2}} & \frac{p_x(v_yp_x-v_xp_y)}{(p_x^2+p_y^2)^{3/2}} & \frac{p_x}{\sqrt{p_x^2+p_y^2} } & \frac{p_y}{\sqrt{p_x^2+p_y^2}}\end{bmatrix}Cj​=⎣⎢⎢⎡​px2​+py2​​px​​−px2​+py2​py​​(px2​+py2​)3/2py​(vx​py​−vy​px​)​​px2​+py2​​py​​px2​+py2​px​​(px2​+py2​)3/2px​(vy​px​−vx​py​)​​00px2​+py2​​px​​​00px2​+py2​​py​​​⎦⎥⎥⎤​

We consider kinematic model with acceleration:

{px=px+dtvx+axdt22py=py+dtvy+aydt22vx=vx+dtaxvy=vy+dtay\begin{cases}p_x = p_x + dtv_x + \frac{a_xdt^2}{2} \\ p_y = p_y + dtv_y + \frac{a_ydt^2}{2} \\ v_x = v_x + dta_x \\ v_y = v_y + dta_y\end{cases}⎩⎪⎪⎪⎪⎨⎪⎪⎪⎪⎧​px​=px​+dtvx​+2ax​dt2​py​=py​+dtvy​+2ay​dt2​vx​=vx​+dtax​vy​=vy​+dtay​​

w=[axdt22aydt22dtaxdtay]w = \begin{bmatrix} \frac{a_xdt^2}{2} \\ \frac{a_ydt^2}{2} \\ dta_x \\ dta_y \end{bmatrix}w=⎣⎢⎢⎡​2ax​dt2​2ay​dt2​dtax​dtay​​⎦⎥⎥⎤​
